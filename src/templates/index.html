WSC API

The WSC API exists for users to create their own clients to services. Registering and loging in to a new account will create a token you can use to verify login to your own application. A responce code of 0 will return a json object and a cookie with the token. A responce of -1 indicates a failed attempt.

    
    GET - Endpoints
    34.42.125.145:8080/api/home
    responce = {
        "code" : 0,
        "body" : jsonObject
    }

    POST - Register new user
    34.42.125.145:8080/api/register
    json = {
        "service" : "service1",
        "username" : "yourchoice",
        "password" : "yourchoice"
    }

    POST - Login
    34.42.125.145:8080/api/login
    json = {
        "service" : "service1",
        "username" : "yourchoice",
        "password" : "yourchoice"
    }

    POST - Update username
    34.42.125.145:8080/api/update/username
    json = {
        "username" : "existingusername",
        "password" : "yourpassword",
        "new_username" : "yourchoice"
    }

    POST - Update password
    34.42.125.145:8080/api/update/password
    json = {
        "username" : "existingusername",
        "password" : "yourpassword",
        "new_password" : "yourchoice"
    }

    POST - Delete user
    34.42.125.145:8080/api/delete
    json = {
        "username" : "existingusername",
        "password" : "yourpassword",
    }

    GET - Logout
    34.42.125.145:8080/api/logout
